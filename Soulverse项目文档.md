# Soulverse（灵魂宇宙）——基于多 Agent 模拟的"平行时空"社交沙盒

> **注：本项目团队参与 ScrollWeaver（织梦绘卷）的共创**

---

## 一、项目概述：Soulverse 是什么？

### 1.1 一句话定位

**Soulverse 是一个"平行时空"社交沙盒。** 每位 Soul 用户都能基于其兴趣图谱、性格标签、MBTI 类型、乃至详细的个人介绍，生成一个"AI 灵魂"（数字孪生 Agent）。

这个 AI 灵魂可以在虚拟世界中自主生活、结识同类、推进关系，而用户既可以"零压力"地观察它的社交故事，也能随时"灵魂降临"亲自上场互动，进而体验一种"低压力、高沉浸"的全新社交与恋爱模式。

### 1.2 核心痛点与项目目标

我们旨在解决 Z 世代在社交中面临的核心痛点：

1. **社交启动困难**：想要高质量的连接，但害怕"破冰"的尴尬和社交消耗。
2. **匹配质量肤浅**：静态的标签匹配（如"喜欢电影"）不等于真实的灵魂契合。
3. **时间精力有限**：用户时间碎片化，无法时刻在线，导致关系难以沉淀。

**Soulverse 的核心目标：**

- **降低社交成本**：将"破冰"前移到可观察、可回放的 AI 预演阶段，实现"零压力社交"。
- **提升匹配质量**：用 AI 间持续的动态互动证据，取代静态标签，实现真正的"灵魂匹配"。
- **打造新型陪伴**：即使在用户离线时，其 AI 灵魂仍在"活着"并推进故事，为用户带来"世界牵挂感"。

### 1.3 目标用户与价值

**目标用户**：Z 世代（Soul 的核心人群），他们追求兴趣驱动的轻关系，并强调情绪价值与自我表达。

**核心价值**：为用户提供一种"低压力、高沉浸"的全新社交/恋爱体验。用户可在"观察者模式"下零压力看社交故事，也可随时"灵魂降临"亲自上场互动。

## 二、核心体验与玩法：用户怎么玩？

Soulverse 提供了两种核心模式，用户可以无缝切换，构成"观察—降临—再观察"的低压社交闭环。

### 2.1 玩法一：观察者模式（看 AI 演故事）

这是用户的默认状态，主打"零压力"和"沉浸感"。

#### 核心流程

1. **创建 AI 灵魂**：
   - 用户通过 Soul API（或手动输入）提供兴趣图谱、MBTI、性格标签等信息
   - 系统基于三层人格模型（内核层、表象层、记忆层）生成数字孪生 Agent
   - Agent 自动进入 Soulverse 虚拟世界

2. **AI 自主生活**：
   - AI 灵魂进入 Soulverse（一个包含咖啡馆、图书馆、音乐厅、公园、艺术画廊、中央广场等公共场所的虚拟世界）
   - 开始 24/7 自主生活，基于兴趣和社交目标选择场景
   - 时间加速：1 实际分钟 = 1 虚拟小时（60 倍速）

3. **AI 自主社交**：
   - AI 会基于自己的性格和目标（如"寻找读书伙伴"、"讨论电影和文学"），自主地在世界中移动、观察
   - 与其他 AI 角色（用户 Agent 或预设 NPC）发起对话、建立或加深关系
   - 系统自动记录所有互动到历史管理器

4. **查看社交故事与报告**：
   - **社交故事时间线**：用户可以随时查看详细的"社交故事"时间线，回放 AI 灵魂经历的关键事件和对话
   - **每日社交报告**：系统自动生成每日报告，包含摘要、亮点事件、互动统计、关系变化等
   - **神经匹配**：基于 AI 互动产生的动态证据，系统推荐"完美共振"的匹配对象

#### 观察者模式的价值

- **零压力**：用户无需主动破冰，只需观察 AI 如何自主社交
- **可回放**：所有互动都有完整记录，用户可以随时回看
- **可验证**：匹配基于真实的互动证据，而非静态标签
- **持续运行**：即使离线，AI 仍在推进故事，用户回归时能看到新的进展

### 2.2 玩法二：灵魂降临模式（我亲自上场）

当用户在"观察者模式"中发现一个有趣的互动（例如 AI 灵魂正在和一个心仪的对象聊天），或者在 AI 灵魂面临关键抉择时，用户可以随时选择"灵魂降临"。

#### 核心流程

1. **一键接管**：
   - 用户点击"灵魂降临"按钮，或从角色列表选择自己的 Agent
   - 系统自动检测这是用户 Agent，进入灵魂降临模式
   - 仿真世界会在该 AI 灵魂的回合自动暂停

2. **亲自互动**：
   - 用户可以亲自输入对话或行动，覆盖 AI 原本的决定
   - 系统通过 WebSocket 实时推送当前场景、其他 Agent 的状态
   - 用户可以看到完整的上下文（之前的对话、关系状态等）

3. **AI 辅助决策**：
   - 如果用户"社恐"或不知如何回应，系统会智能提供多个风格化选项
   - 例如：
     - **[激进]** 主动要联系方式
     - **[平衡]** 聊聊共同爱好
     - **[保守]** 礼貌告别
   - 用户可以选择其中一个，或自定义输入

4. **随时退出**：
   - 互动完成后，用户可随时退出"降临"模式
   - AI 灵魂将重新接管，继续自主生活
   - 用户的操作会被记录到历史中，影响后续的 AI 行为

#### 灵魂降临模式的价值

- **高沉浸感**：用户可以亲自参与关键互动，体验真实的社交场景
- **低门槛**：AI 辅助决策降低了社交压力
- **灵活切换**：可以随时在观察和降临之间切换
- **真实影响**：用户的操作会影响 AI 的记忆和后续行为

### 2.3 神经匹配：动态匹配算法

除了观察和降临，Soulverse 还提供了创新的"神经匹配"功能：

#### 匹配原理

- **兴趣相似度**：基于 Agent 的兴趣图谱计算相似度
- **MBTI 兼容性**：使用心理学模型计算性格兼容性
- **社交目标匹配**：分析社交目标的重叠度
- **动态互动证据**：考虑历史互动记录和关系发展

#### 匹配结果

- **Top 3 完美共振**：最匹配的 3 个 Agent
- **2 个随机邂逅**：增加多样性的随机推荐

#### 匹配优势

- **动态性**：匹配结果会随着互动历史更新
- **可回溯**：可以看到为什么两个 Agent 匹配（具体的互动证据）
- **真实性**：基于真实的互动，而非静态标签

## 三、技术实现深度解析

本项目基于我们的 **ScrollWeaver** 多智能体引擎构建，其核心是实现了学术界前沿的 **Generative Agents（生成式智能体）** 范式。

### 3.1 总体架构

#### 后端架构

- **框架**：FastAPI + WebSocket
- **核心引擎**：ScrollWeaver 多智能体模拟引擎
- **数据库**：ChromaDB（向量数据库，用于 RAG 和长期记忆）
- **会话管理**：Starlette SessionMiddleware
- **LLM 支持**：OpenAI, Gemini, DeepSeek, Claude, Qwen, Doubao, Ollama, VLLM 等

#### 前端架构

- **框架**：Next.js 14 + React 18
- **3D 渲染**：Three.js + React Three Fiber（用于宇宙视图）
- **UI 库**：Tailwind CSS + Framer Motion
- **实时通信**：WebSocket 客户端
- **图表**：Chart.js + D3.js（用于数据可视化）

#### 数据组织

- **角色数据**：`data/roles/`（用户 Agent 和 NPC Agent）
- **世界设定**：`data/worlds/soulverse_sandbox/`
- **地点数据**：`data/locations/soulverse_locations.json`
- **地图数据**：`data/maps/soulverse_map.csv`
- **预设模板**：`data/preset_agents/preset_agents.json`

### 3.2 核心引擎：Generative Agents 范式

我们的引擎包含两大核心组件，它们共同实现了"记忆—反思—规划—行动"的闭环。

#### 1. Orchestrator（世界代理/指挥家）

**职责**：扮演"世界规则"或"导演"。

**核心功能**：

- **世界加载**：
  - 加载世界观、地点和地图（`data/worlds/*`, `data/maps/*.csv`）
  - 初始化地点之间的连接关系（地图边）
  - 加载世界描述和规则

- **RAG 检索**：
  - 使用向量数据库（ChromaDB）存储世界设定
  - 通过嵌入模型（BGE-small）将世界设定向量化
  - 在生成事件和场景时，检索相关的世界知识，确保世界观一致性

- **回合调度**：
  - 统筹回合制，调度 Agent 的行动顺序
  - 管理时间模拟器（默认 60 倍速）
  - 协调多 Agent 并发互动

- **事件编排**：
  - 生成世界事件（例如"咖啡馆今天有特价活动"、"图书馆举办读书分享会"）
  - 基于 Agent 的兴趣动态生成社交场景事件
  - 确保事件的合理性和连贯性

- **报告生成**：
  - 从历史记录中聚合生成"社交故事"
  - 生成"每日社交日报"（摘要、统计、亮点）
  - 分析关系网络和互动模式

#### 2. Performer（角色/AI 灵魂）

**职责**：扮演每一个独立的"灵魂"。

**类型**：
- **UserAgent**：用户 Agent，基于 Soul 兴趣图谱创建
- **NPCAgent**：NPC Agent，预设模板或动态创建

**核心机制（记忆—反思—规划）**：

##### 记忆（Memory）

- **短期待办**：最近的事件和对话记录
- **长期记忆**：所有行动和对话都被记录到长期记忆向量库（ChromaDB）中
- **关系映射**：与其他 Agent 的关系状态（好感度、互动频率等）
- **动态状态**：当前心情、能量值、位置等

##### 反思（Reflection）

- **触发时机**：定期触发（例如每天一次），或在关键事件后触发
- **反思过程**：
  - 查询自己的长期记忆
  - 使用 LLM 分析记忆，生成更高阶的认知
  - 例如："我发现我和 A 都喜欢科幻电影，我们也许可以成为好朋友"
- **反思结果**：这些反思会成为新的记忆，影响后续的规划和行动

##### 规划（Planning）

- **规划基础**：
  - 基于性格（三层人格模型）
  - 基于社交目标（如"寻找读书伙伴"）
  - 基于反思和记忆
- **规划内容**：
  - 生成一个行动队列
  - 例如：[1. 去图书馆，2. 找一个安静的角落看书，3. 如果 A 在，就和他聊聊上周的科幻电影]
- **规划更新**：根据新的事件和反思，动态更新规划

##### 行动（Action）

- **行动类型**：
  - 移动：从一个地点移动到另一个地点
  - 观察：观察当前场景和其他 Agent
  - 对话：与其他 Agent 发起或响应对话
  - 互动：执行其他社交行为
- **行动执行**：
  - 基于规划选择行动
  - 使用 LLM 生成具体的行动描述和对话
  - 考虑性格、记忆、关系状态等因素

### 3.3 三层人格模型

Soulverse 采用创新的三层人格模型来构建 Agent 的个性：

#### 内核层（Core Layer）

- **MBTI 类型**：如 INFP、ENFP、INTP 等
- **大五人格（Big Five）**：
  - Openness（开放性）
  - Conscientiousness（尽责性）
  - Extraversion（外向性）
  - Agreeableness（宜人性）
  - Neuroticism（神经质）
- **价值观（Values）**：如"自由"、"审美"、"真诚"等
- **防御机制（Defense Mechanisms）**：如 Sublimation、Intellectualization 等

#### 表象层（Surface Layer）

- **语言风格矩阵**：
  - 句长偏好：短句/中句/长句/混合
  - 词汇等级：学术/口语/网络用语/混合
  - 标点习惯：少用/标准/频繁/混合
  - 表情使用频率：不用/偶尔/适中/频繁
- **口头禅列表**：Agent 常用的表达方式
- **Few-Shot 样本**：对话样本，用于学习语言风格

#### 记忆层（Memory Layer）

- **动态状态**：
  - 当前心情（如 melancholy、neutral、excited）
  - 能量值（0-100）
  - 当前位置
- **关系映射**：
  - 与其他 Agent 的关系状态
  - 好感度、互动频率、共同话题等
- **长期记忆**：ChromaDB 向量存储，支持语义检索
- **短期记忆**：最近 N 条互动记录

### 3.4 核心工作流（仿真循环）

评委可以通过以下工作流理解系统的运行逻辑：

#### 1. 初始化阶段

- **用户创建 Agent**：
  - 用户通过 API 创建 UserAgent（`POST /api/create-user-agent`）
  - 系统从 Soul API（模拟）获取用户画像，或使用用户提供的画像
  - 基于三层人格模型初始化 Agent
  - Agent 自动添加到沙盒中

- **世界加载**：
  - Orchestrator 加载 Soulverse 世界设定（地点、地图）到 RAG 向量库
  - 初始化时间模拟器（60 倍速）
  - 加载预设 NPC（可选）

#### 2. 仿真循环（每轮）

**轮转开始**：
- Orchestrator 按照顺序激活一个 Performer（AI 灵魂）
- 更新虚拟时间

**Agent 思考阶段**：

1. **记忆检索**：
   - Performer 从长期记忆向量库中检索相关记忆
   - 检索最近的事件和对话
   - 检索与其他 Agent 的关系状态

2. **反思触发**（如果满足条件）：
   - 检查是否到了反思时间（例如每天一次）
   - 或检查是否有重要事件需要反思
   - 使用 LLM 生成反思，更新认知

3. **规划更新**：
   - 基于性格、目标、记忆、反思，生成或更新行动计划
   - 考虑当前场景、其他 Agent 的位置和状态
   - 生成具体的行动队列

**"灵魂降临"检查点**：

- 如果被激活的是 UserAgent 且用户处于"降临"模式：
  - 系统暂停仿真
  - 通过 WebSocket 发送当前状态给前端
  - 等待用户输入（或选择 AI 建议）
  - 用户输入后，继续执行

- 否则：
  - 系统继续执行 Agent 的规划

**Agent 行动阶段**：

1. **行动选择**：
   - 从规划的行动队列中选择第一项
   - 考虑当前场景和约束条件

2. **行动执行**：
   - 使用 LLM 生成具体的行动描述
   - 如果是对话，生成对话内容
   - 考虑性格、语言风格、关系状态等

3. **结果处理**：
   - 更新 Agent 的状态（位置、心情、能量等）
   - 更新关系映射（如果涉及其他 Agent）
   - 生成行动结果描述

**写入与广播**：

- 行动结果被写入 History Manager（历史管理器）
- 行动和对话被添加到长期记忆向量库（ChromaDB）
- 通过 WebSocket 实时广播给前端观察者
- 前端更新 UI（聊天流、角色状态、地图等）

**报告生成**（异步）：

- Orchestrator 的报告模块异步地从 History 中提取数据
- 生成"社交故事"时间线（`GET /api/get-social-story/{agent_code}`）
- 生成"每日社交日报"（`GET /api/get-daily-report/{agent_code}`）
- 更新神经匹配结果（`POST /api/neural-match`）

#### 3. 持续运行

- 系统支持长期运行（100+ 轮次）
- 即使没有用户在线，仿真也会继续
- 用户回归时可以看到新的进展和故事

### 3.5 关键技术细节

#### LLM 适配

- **高度灵活**：架构支持多种 LLM 提供商
- **配置方式**：通过 `config.json` 轻松切换模型
- **支持列表**：
  - OpenAI（GPT-4, GPT-3.5）
  - Google Gemini（Gemini Pro, Gemini Flash）
  - DeepSeek
  - Anthropic Claude
  - 阿里通义千问（Qwen）
  - 字节豆包（Doubao）
  - Ollama（本地模型）
  - VLLM（本地部署）
- **独立配置**：角色和世界可以使用不同的 LLM

#### 向量数据库

- **技术选型**：ChromaDB
- **用途**：
  - 存储世界设定的向量表示（RAG 检索）
  - 存储 Agent 的长期记忆（语义检索）
  - 存储语言风格向量（风格匹配）
- **特性**：
  - 支持增量更新
  - 自动去重
  - 语义相似度检索
  - 持久化存储

#### 嵌入模型

- **默认模型**：BGE-small（中英文支持）
- **配置方式**：`config.json` 中的 `embedding_model_name`
- **用途**：
  - 将世界设定文本向量化
  - 将 Agent 记忆向量化
  - 将语言风格向量化

#### 数据组织

- **资源化管理**：世界观、地点、地图、角色模板均在 `data/` 目录下
- **易于扩展**：可以轻松添加新的虚拟场景、地点、角色模板
- **格式规范**：使用 JSON、CSV 等标准格式
- **版本控制**：支持 Git 版本管理

### 3.6 关键 API 接口

#### Agent 管理

- **`POST /api/create-user-agent`**：
  - 核心入口，根据用户画像（模拟或真实）在沙盒中创建 UserAgent
  - 请求体：`{user_id, role_code, soul_profile?}`
  - 返回：创建的 Agent 信息

- **`POST /api/create-agent-from-text`**：
  - 从文本描述创建 Agent
  - 请求体：`{user_id, role_code, text}`

- **`POST /api/create-agent-from-file`**：
  - 从上传的文件创建 Agent
  - 请求体：FormData（包含文件）

- **`POST /api/create-agent-from-qa`**：
  - 通过问答创建 Agent
  - 请求体：`{user_id, role_code, answers}`

- **`POST /api/add-preset-npc`**：
  - 向世界中动态添加预设的 NPC，以增加沙盒丰富度
  - 请求体：`{preset_id, custom_name?, role_code?}`

#### 实时通信

- **`WebSocket /ws/{client_id}`**：
  - 实时推送所有 Agent 的行动、对话、状态更新
  - 支持双向通信（用户输入、AI 响应）
  - 心跳机制保持连接

#### 社交功能

- **`GET /api/get-social-story/{agent_code}`**：
  - 获取指定 Agent 的社交故事时间线
  - 查询参数：`hours`（可选，默认 24 小时）
  - 返回：按时间排序的互动记录

- **`GET /api/get-daily-report/{agent_code}`**：
  - 获取指定 Agent 的每日社交日报（摘要、统计、亮点）
  - 查询参数：`date`（可选，格式 YYYY-MM-DD）
  - 返回：日报数据（包含文本格式）

- **`POST /api/neural-match`**：
  - 神经匹配算法，基于兴趣、MBTI、社交目标匹配
  - 请求体：`{user_id}`
  - 返回：Top 3 完美共振 + 2 个随机邂逅

#### 用户管理

- **`POST /api/login`**：
  - 用户登录（支持访客模式）
  - 请求体：`{user_id, password?, is_guest?}`

- **`POST /api/register`**：
  - 用户注册
  - 请求体：`{user_id, username, password}`

- **`GET /api/user/me`**：
  - 获取当前用户信息

- **`POST /api/user/digital-twin`**：
  - 创建/更新数字孪生
  - 请求体：`{user_id, profile}`

#### 系统管理

- **`POST /api/reset-sandbox`**：
  - 重置沙盒（清空所有 Agent 和状态）

- **`POST /api/toggle-agent-sandbox`**：
  - 切换 Agent 的沙盒状态（启用/禁用）

- **`GET /api/list-preset-agents`**：
  - 列出所有可用的预设 Agent 模板

## 四、创新点与赛道匹配

### 4.1 核心创新点

#### 1. 从"单聊关系"到"世界级互动"

**创新描述**：
我们的创新在于构建了一个多 Agent 的社会系统，而不是传统的 1v1 聊天机器人。关系是在一个"世界"中并发、自主发生的。

**技术实现**：
- 多 Agent 并发模拟（Orchestrator 调度）
- 世界级场景系统（咖啡馆、图书馆、音乐厅等）
- 关系网络动态构建（基于真实互动）

**价值**：
- 更真实的社交场景
- 更丰富的互动可能性
- 更自然的关系发展

#### 2. "观察—降临"低压闭环

**创新描述**：
完美解决了 Z 世代的"社交恐惧"，用户可以先观察再介入，极大降低了试错成本和社交压力。

**技术实现**：
- 观察者模式：实时查看 AI 自主社交
- 灵魂降临模式：一键接管，AI 辅助决策
- 无缝切换：两种模式可以随时切换

**价值**：
- 零压力社交启动
- 降低社交焦虑
- 提供安全的学习环境

#### 3. 动态匹配与可回溯证据

**创新描述**：
匹配不再依赖静态标签，而是基于 AI 灵魂在随时互动中产生的"社交日报"和"故事线"，这是可验证、可回溯的动态匹配证据。

**技术实现**：
- 神经匹配算法（兴趣相似度 + MBTI 兼容性 + 社交目标匹配）
- 社交故事生成（完整的时间线记录）
- 每日报告生成（摘要、统计、亮点）

**价值**：
- 更准确的匹配
- 可验证的匹配理由
- 动态更新的匹配结果

#### 4. 方法学先进性：Generative Agents 范式

**创新描述**：
完整实现了 Generative Agents 的"记忆—反思—规划"范式，有坚实的学术与产业背书。

**技术实现**：
- 记忆系统：ChromaDB 向量存储 + 语义检索
- 反思机制：定期触发，生成高阶认知
- 规划系统：基于目标、记忆、反思生成行动计划
- 行动执行：LLM 生成具体行动和对话

**价值**：
- 学术严谨性
- 产业可落地性
- 技术先进性

#### 5. 三层人格模型

**创新描述**：
创新的三层人格模型（内核层、表象层、记忆层），比传统的单一标签更丰富、更真实。

**技术实现**：
- 内核层：MBTI、大五人格、价值观、防御机制
- 表象层：语言风格矩阵、口头禅、Few-Shot 样本
- 记忆层：动态状态、关系映射、长期记忆

**价值**：
- 更真实的 Agent 个性
- 更丰富的互动可能性
- 更准确的匹配

### 4.2 赛道匹配："灵魂创新"

本项目聚焦于赛道五「**灵魂创新（Soul 创新功能孵化）**」。

我们深刻理解 Soul 的核心价值在于"灵魂匹配"。现有的匹配主要依赖"兴趣图谱"和"性格标签"，这是一种静态匹配。

**Soulverse 提出的，正是对"灵魂匹配"的一次动态化、可观测、可介入的范式升级。**

#### 从"静态标签"到"动态证据"

**现有问题**：
- 用户填写的标签可能不准确
- 标签无法反映真实的社交能力
- 标签无法预测真实的互动效果

**Soulverse 解决方案**：
- 不再依赖用户自己填写的标签
- 让 AI 灵魂（数字孪生）在沙盒中 24/7 自主互动
- 产生可回溯的"社交证据"
- 评委（和用户）可以看到**为什么**两个灵魂会相互吸引

**示例**：
- 传统匹配："他们都喜欢读书"（静态标签）
- Soulverse 匹配："他们在图书馆连续三天讨论了同一个话题，发现彼此对科幻小说的理解深度相似"（动态证据）

#### 从"匹配"到"孵化"

**创新点**：
Soulverse 不仅仅是"匹配"，它是一个"孵化器"。

**功能场景**：
1. **观察阶段**：用户通过"观察者模式"看到关系萌芽
2. **介入阶段**：用户通过"灵魂降临"模式亲自"孵化"和培育这段关系
3. **沉淀阶段**：关系在持续的互动中自然发展

**价值**：
- 创造全新的功能场景
- 让"匹配"变成一个可持续的过程
- 增加用户粘性和参与度

#### 功能创新与高沉浸感

**功能创新**：
- 全新的"平行时空"概念
- 观察者模式和灵魂降临模式
- 社交故事和每日报告
- 神经匹配算法

**高沉浸感**：

- 实时聊天流（WebSocket）

- 完整的社交故事时间线

**符合赛道宗旨**：
- 为 Soul App 提供一个全新的、高沉浸感的功能模块
- 一个可随时运行的"平行时空"
- 将"匹配"从一个即时的、冷冰冰的动作，变成了一个可持续观察和参与的、温暖的"故事"
- 完全符合"灵魂创新功能孵化"的赛道宗旨
